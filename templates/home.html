Why I love Express

Express is the most reliable, beautiful thing in the JavaScript world.

It is so ubiquitous, almost every tech giant from Netflix to Nasa makes use of it at some level of their stack.

I’ve been using it for over 7+ years and have built and operated many projects with it, including Borderline.biz and BoredReading.com. To me, Express is the Toyota Corolla of JS Frameworks. It’s boring, never really changes, it’s just kinda there, day after day, it does what it says on the box... and just keeps chugging along, like a champ, without fail.

I love express. I really do. ❤️

Frequently Asked Questions

What is Express?

Express.js is a minimal and flexible Node.js web application framework that provides a robust set of features for web and mobile applications.

How is Express different from React/Next/Angular/etc?

Express renders HTML on the backend using templates and serves it to the frontend. It’s a different paradigm to React, where a large portion of the DOM is rendered on the client.

In simpler terms, if you’re application has a lot of pages/navigational complexities, you might find Express solves those problems better. For instance, if you were making a CRM, Job board, Ecommerce platform, you might find Express works really well.

 However, if you have very few pages, and most of your app is very reactive and updates instantly on the DOM based on user input, you should look into other boilerplates that use React. For example, if you’re building the next Gmail or WhatApp, you’ll need the live reactivity of React.
